#Color Code Cosme 色からコスメを探せる、
データーベース型のwebアプリ。
検索ページをアップデート。

Overview概要

このアプリは、自分に合うコスメを色で検索できるデータベース型のWebアプリです。
写真から色を抽出、カラーピッカーで色を選ぶ、カラーコードを入力するという３つの検索方法があり、好きな色のコスメアイテムを見つけられるようになっています。膨大なデータベースの中から、ブランドやカテゴリを横断して比較できるため、自分にあったアイテムを効率的に探すことが可能です。

  • ● 業界：コスメ
  • ● ボリューム：1ページ
    • ● ターゲット：
  • ・10〜30代のコスメを探している女性
  • ・芸能人などが使用しているコスメと

  • ● ツール：Figma
  • ● 言語：HTML、CSS、JavaScript
  • ● 期間：
    ・打ち合わせ 1週間
    ・デザイン 1週間
    ・コーディング 1週間

Solutionsリニューアル前の課題と改善

このアプリでは、3種類の検索方法が1ページにまとまっており、リニューアル前はテキストのみで使い方を解説していたため、検索の流れが分かりづらく、初見のユーザーにとって操作が直感的ではないという課題がありました。さらに、なるべくサイトの構成などは変更せず、ビジュアルのみで改善をご希望との事でした。
そこで、視認性と操作性を向上させ、検索の流れが文字だけでなく視覚的に伝わるよう設計を見直しました。また、使用媒体の8割がスマホであるため、スマホでの操作性を最優先に考えたデザインへと最適化。元の要件を維持しつつ、ユーザーが迷わずに使える導線設計を強化しました。

デザイン面では、好きな色を自由に選べるアプリの特性上、サイト自体のカラーが検索体験を邪魔しないようにモノトーンの配色を維持しつつ、リンクの視認性が低いという課題があったため、リンク部分は一般的に認識されやすい青系色を採用。直感的に識別できるよう改善しました。また、ユーザー層が幅広いため、個性的なデザインではなく、シンプルでベーシックなUIを採用し、誰もが快適に使える設計を意識しています。

Processプロセス

1. リサーチ

ユーザーの使い方を把握

アプリのリサーチプロセスでは、ユーザー行動を深く理解するために、まず、SNSなどを見て、実際にどのような使い方をしているかを確認。特に、アイドルの写真から他アイテムを探したり、自分好みの色味のリップを探したりするなどの途多いことがわかりました。他にも、質問サイトなどでのやり取りからも得られる情報を活かし、具体的なターゲットユーザーや使い方をイメージしながらデザインに落とし込むようにしています。
また、スマホでの使用時に、画像から十字キーでの色抽出が難しいという課題があることが判明しました。そのため、一部のユーザーは、色抽出の際にスポイトツールと組み合わせて使用していることがわかり、この部分はリニューアル後も使用できるようにしています。

2. デザインリニューアル

テキストを読まなくても、分かる、使える画面に。

デザインリニューアルでは、ユーザーが直感的に機能や構造を理解できるよう、構成を大幅に見直し、ほとんどテキストを読まなくても問題なく使えるUIへとリニューアルしています。
リニューアル前は主にテキストで説明されていた部分を、視覚的な要素を活かして、ファーストビューで機能がわかるレイアウトに変更。
特に、色検索に関する要素が邪魔にならないように、配色はモノトーンを維持し、導線部分には一般的にリンクとして認識しやすい青色を採用しています。ベーシックなデザインにしたかったので、最初はGoogleで一般的に使用されている青色を検討しましたが、ユーザー層として厚い10~30代女子の反応を考え、大手ファッションサイトのZOZOTOWNを参考に少し可愛らしい水色に変更しています。この色は、全体のデザインに優しさと親しみやすさを加え、視覚的にも心地よい印象を与えることができると考えました。また、リンクとしての識別性を保ちつつ、よりユーザーにとって使いやすく、親しみやすい色味に調整しています。

短めで、読みやすい文言を使い認知しやすく。

余白を少なめにし、１画面に２ツールが 収まるように。

リサーチを踏まえ、既存の機能性を維持

既存の機能も損なわないようにリニューアルしています。画像からの色抽出機能を使う際にスポイトツールとの併用使用ができるよう、距離感に配慮し、画面内に画像とツールが収まるように余白を調整しました。
さらに、「色抽出が難しい時は」というヘルプページへの導線を、クエスチョンアイコンを使用して目立たせ、初めてのユーザにも、この使い方を認知しやすくしています。

を押下すると開く

使用頻度の高い機能は、アクセスしやすく

検索の絞り込み機能については、これまで検索セクションの下に配置されていましたが、ページのどの場所からでもアクセスできるよう、追従する絞り込みボタンを設置し、モーダル表示に変更しました。 また、検索結果とプレビューも常に確認できるよう、フッターに固定し、使いやすさを向上させました。

大手ECサイトを参考に、自然な視線誘導

検索結果についても、表示される商品画像が小さく見づらかったため、Amazonや楽天市場、Yahoo!などの大手ECサイトを参考に、商品の画像を大きく表示するように変更。
また、商品名やブランド名が視認性に欠けていたため、ジャンプ率を調整し、商品画像→商品名→タグ、ブランド名やカラー名と、情報が整理されて見やすくなるように順位付け、ユーザーが商品を見つけやすくなるようなレイアウトにしています。

3. コーディング

実装時のコストを最小限に抑えたコーディング

コーディングでは、主にフロントエンドの実装を担当しました。リニューアル後のデザインに合わせて、既存の検索ロジックをなるべくそのまま流用できるようにコーディングしています。
また、従来のシステムでは、検索を実行した際に、ページの最上部に戻され、ユーザーが自力で結果表示されている部分を探さなければならないという不便さがありました。そのため、ユーザーが検索結果にスムーズにアクセスできるように、オートスクロール機能を実装。この機能により、検索後にページが自動的に結果表示部分までスクロールし、ユーザーの操作負担を軽減しています。
